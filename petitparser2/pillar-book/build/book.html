PetitParser2 and Bounded Seas

In this tutorial we describe how to quickly build a parser to extract a javascript code from any HTML source.

We show how can we utilize bounded seas - a tool that allows us to focus on the interesting parts of an input (i.e. javascript in our case) and ignore the rest (the remaining HTML code) to save time and obtain results as fast as possible.

We also describe performance optimizations abilities of PetitParser, that will allow us to generate high-performance parsers from our grammars that have performance comarable to table-driven parsers such as SmaCC or hand-written parsers such as RBParser.

Structure

In Extracting Javascript we start with a simple script. Later in Real Grammar, we turn the script into a proper grammar and add tests. In Extracting Structure, we extend the parser to extract an html structure and we add tolerance for malformed documents.

In Optimizations we inspect optimizations capabilities of PetitParser2 and in Memoizations we describe tooling of PetitParser2 that will help us to pinpoint the perofrmance bottlenecks and we show some techniques how to fix them.
